## A. 上手 / 首次設定
- [ ] 🔴 **Claude Token 無引導** — 只寫 `claude setup-token 取得`，沒有「怎麼拿」連結或一鍵流程。
- [ ] 🔴 **「留空也能用」太隱晦** — Token 欄看起來必填，新使用者直接被勸退。
- [ ] 🔴 **首次設定後沒有「下一步」引導** — 進儀表板只看到「尚無錄音紀錄」，缺 onboarding 帶我錄第一段。
- [ ] 🟢 **預設路徑只顯示、不可一鍵採用** — 得自己理解「留空＝用這個」。

## B. 名詞與文案（主要影響非技術使用者）
- [ ] 🔴 **工程名詞直接外露** — `whisper / stt_worker / llm_service / openclaw`、Docker、WSL、「未建立」等未翻成人話或隱藏。
- [ ] 🟡 **「自動」與實際「手動逐檔」矛盾** — 錄音頁標題寫「完成後自動轉譯並生成」，操作卻要逐筆按。
- [ ] 🟡 **狀態用詞不一致** — 「轉會議紀錄」vs 徽章「待整理／整理中」是同件事卻不同詞；「轉逐字稿」vs「轉譯中」。
- [ ] 🟡 **危險操作前後說法打架** — 說明「刪除…與處理紀錄」、確認框卻說「逐字稿與會議紀錄不受影響」。
- [ ] 🟢 **關閉確認列出一串服務名稱** — 嚇到只想關視窗的人。
- [ ] 🟢 **「Whisper API Key（目前未使用）」** — 未使用就不該外露。

## C. 流程 / 狀態機
- [x] 🔴 **兩段式手動流程不直觀** — `待轉譯→轉譯中→待整理→完成` 每態要按不同鈕，缺可視步驟條。 〔已完成〕
- [x] 🟡 **auto / manual 模式不透明** — `config.pipeline` 有 `mode/auto`，但 UI 無切換、也不顯示目前模式。 〔已完成〕
- [x] 🟡 **錄音前「會議主題／日期」缺說明** — 沒填會怎樣、檔名規則為何，無提示。 〔已完成〕
- [x] 🟢 **「手動切割」剪刀鈕無說明** — 使用者不敢按。 〔已完成〕

## D. 失敗處理與錯誤可視化（技術使用者頭號痛點）
- [x] 🔴 **清單沒有「失敗」狀態** — `REC_STAGE` 無 failed，轉譯失敗默默退回「待轉譯」，跟沒按一樣。 〔已完成〕
- [x] 🔴 **「轉會議紀錄」失敗靜默** — 背景生成失敗後默默回到「待整理」，無錯誤 toast / 無紅標。 〔已完成〕
- [x] 🔴 **`callApi` 把錯誤吞成 `null`** — 上層只給通用 toast；錯誤處理不一致（部分 handler 才讀 `r.error.message`）。 〔已完成〕
- [x] 🟡 **讀取類錯誤含糊** — `viewResult/get_transcript` 失敗一律「尚未生成」，分不出「沒跑/跑掛/LLM 報錯」。 〔已完成〕
- [x] 🟡 **無重試入口** — 失敗後沒有「重試」按鈕，只能盲按原按鈕。 〔已完成〕
- [x] 🟡 **無法取消進行中的工作** — 轉譯/整理一旦開始不能 abort。 〔已完成〕

## E. 可觀測性 / 日誌 / 診斷
- [x] 🔴 **UI 內無任何 Log 檢視** — 有 `worker.log`、容器有 `docker logs`，但畫面看不到，出事得離開 app 翻檔。 〔已完成〕
- [ ] 🟡 **服務「異常」無 detail** — 徽章點下去沒有 exit code / 最後錯誤 / log 展開。
- [ ] 🟡 **看不出實際後端** — docker 還是 native？哪個 whisper 模型、哪個 claude model，UI 不顯示。
- [ ] 🟢 **健康卡「X/Y 運行中」分母語意不清** — native 模式某些服務 absent 時 X/Y 易誤解。

## 建議修復優先序
1. **先救信任**（🔴 D + E）：失敗態、錯誤訊息、log 面板、重試。
2. **再補工作流**（🔴 F + G）：重新生成 + 套用新 prompt、批次處理。
3. **然後降門檻**（🔴 A + B + C）：Token 引導、名詞人話化、auto/manual 講清楚 + 步驟條。
4. **最後打磨**（🟡 / 🟢 H–N）：清單管理、輪詢重構、設定儲存模型、收掉死整合、版面與可及性。
